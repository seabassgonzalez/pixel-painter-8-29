Pixel Painter

Description

You will be created a simple ‘Paint’ app. You will render a grid of 10x10 ‘large pixels’, which can be individual filled with color to create simple images. You should focus on the architecture: storing and passing information cleanly between components. In addition, you should watch out for performance, e.g. if you increase the grid size (i.e. the number of pixels) too large, you might even break CodeSandbox, so be careful as you develop.

Features

  1. Create a 10x10 grid of pixels, and an input color field that will contain a color in hex (e.g ‘#ff0000’). Upon clicking on a pixel, it should be filled with the current color.
  2. Each ‘pixel’ should quite large on the page (e.g. 50px by 50px) and have a border, so you can tell it apart.
  3. Clicking on a previously filled pixel should replace the pixel’s color with the current color.
  4. You do not need to use Material UI for this, and can use your own CSS or styles as you see fit.
  5. BONUS: Clicking and holding, should fill all the pixels you mouse over until releasing the mouse.
  6. BONUS: Toggle the borders on pixels on/off (on for helping to creating an image, and off to see the result
  7. BONUS: Add a way to vary the number of pixels, or the actual px size of a pixel.
